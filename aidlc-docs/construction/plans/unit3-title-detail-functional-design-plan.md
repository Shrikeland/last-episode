## Clarifying Questions

### Q1: RatingInput — формат оценки
Требования допускают "целое число или полузвёзды". Какой вариант реализуем в MVP?

A) **Целые числа 1–10** — 10 кнопок/звёзд, только целые значения
B) **Полузвёзды** — 10 звёзд × 2 половины = 20 значений (1.0, 1.5, 2.0…10.0)

[Answer]: B

---

### Q2: NotesEditor — механизм сохранения
Как сохраняются заметки?

A) **Autosave on blur** — сохраняется автоматически при потере фокуса (без кнопки)
B) **Debounce autosave** — сохраняется через 1–2 сек после последнего ввода (без кнопки)
C) **Explicit save button** — только по кнопке "Сохранить"
D) **Debounce + visual indicator** — debounce 1.5 сек + текст "Сохранено" / "Сохраняю..."

[Answer]: B

---

### Q3: Смена статуса на "completed" для сериалов/аниме
Когда пользователь переключает статус на `completed` для tv/anime — что происходит с эпизодами?

A) **Ничего** — эпизоды остаются как есть, статус и прогресс независимы
B) **Автоматически отмечаются все эпизоды** — все `is_watched = true`
C) **Спрашивать пользователя** — диалог "Отметить все эпизоды как просмотренные?"

[Answer]: B

---

### Q4: "Отметить весь сезон" — поведение при повторном нажатии
Если все эпизоды сезона уже отмечены и пользователь снова жмёт "Отметить весь сезон":

A) **Toggle** — снимает отметку со всех эпизодов сезона (кнопка работает как toggle)
B) **Только mark** — кнопка активна только если не все отмечены; исчезает/деактивируется когда все отмечены
C) **Без изменений** — повторное нажатие ничего не делает

[Answer]: A

---

### Q5: Оптимистичные обновления — поведение при ошибке
Если Server Action для отметки эпизода завершился с ошибкой:

A) **Rollback + silent** — UI возвращается к предыдущему состоянию, без уведомления
B) **Rollback + toast** — UI откатывается + показывается toast "Ошибка сохранения"
C) **Retry** — автоматическая повторная попытка 1 раз, при неудаче — rollback + toast

[Answer]: C

---

### Q6: EpisodeRow — дополнительная информация
Что отображается в строке эпизода помимо чекбокса и названия?

A) **Минимум** — только номер эпизода + название + чекбокс
B) **С длительностью** — номер + название + runtime (если есть) + чекбокс
C) **С датой** — номер + название + runtime + дата просмотра (watched_at) + чекбокс

[Answer]: C

### Q7: Навигация назад со страницы тайтла
Куда ведёт кнопка "назад" / breadcrumb на странице `/media/[id]`?

A) **Всегда /library** — фиксированная ссылка на библиотеку
B) **Browser history back** — `router.back()` (возврат на предыдущую страницу)
C) **Breadcrumb** — статичный breadcrumb "Библиотека > Название тайтла"

[Answer]: B
