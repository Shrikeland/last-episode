# Business Rules — Unit 3: Title Detail + Progress

## BR-01: Доступ к тайтлу
- Пользователь может видеть и редактировать только свои тайтлы (RLS на уровне БД)
- Если `media_items.user_id ≠ auth.uid()` — Supabase вернёт пустой результат → `notFound()`
- Server Actions проверяют `auth.getUser()` перед любой операцией

## BR-02: Рейтинг — полузвёзды
- Диапазон: 1.0 до 10.0 включительно, шаг 0.5
- Допустимые значения: 1.0, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0, 4.5, 5.0, 5.5, 6.0, 6.5, 7.0, 7.5, 8.0, 8.5, 9.0, 9.5, 10.0
- Значение `null` — рейтинг не выставлен (допустимо)
- **БД**: `rating NUMERIC(3,1) CHECK (rating >= 1.0 AND rating <= 10.0)` (миграция в Unit 3)
- Повторный клик на текущую оценку → сбрасывает рейтинг в `null`

## BR-03: Статус и автоматическое отмечание эпизодов
- При смене статуса на `completed` для `type = 'tv'` или `type = 'anime'`:
  → все эпизоды отмечаются `is_watched = true`, `watched_at = NOW()`
- При смене статуса на любой другой (не `completed`) — эпизоды **не трогаются**
- Для `type = 'movie'` — статус меняется без каких-либо операций с эпизодами (их нет)

## BR-04: "Отметить весь сезон" — toggle
- Если **не все** эпизоды сезона просмотрены → нажатие отмечает **все** (`is_watched = true`)
- Если **все** эпизоды сезона просмотрены → нажатие снимает отметку со **всех** (`is_watched = false`, `watched_at = NULL`)
- Логика определения: `season.episodes.every(e => e.is_watched)`

## BR-05: Оптимистичные обновления и обработка ошибок
- Для `toggleEpisode` и `markSeason`: применяется оптимистичный UI через `useOptimistic`
- При ошибке Server Action: автоматически 1 повторная попытка (retry)
- Если retry тоже упал: rollback optimistic state + `toast.error("Ошибка сохранения")`
- Для `updateStatus`, `updateRating`, `updateNotes`: без оптимистичного UI (менее критично для UX)

## BR-06: NotesEditor — debounce autosave
- Сохранение инициируется через 1500мс после последнего ввода символа
- При размонтировании компонента (смена страницы) — pending сохранение выполняется немедленно (flush)
- Пустая строка → сохраняется как `null` в БД (не пустая строка)
- Нет отдельной кнопки "Сохранить"

## BR-07: Прогресс — только для tv/anime
- Секция с сезонами/эпизодами отображается **только** для `type = 'tv'` или `type = 'anime'`
- Для `type = 'movie'` — секция прогресса скрыта
- Если тайтл `tv/anime` не имеет сезонов в БД (edge case) — секция скрыта

## BR-08: EpisodeRow — отображение данных
- Всегда показывать: номер эпизода + название + чекбокс
- Показывать runtime только если `episode.runtime_minutes !== null`
- Показывать `watched_at` только если `episode.is_watched === true` И `episode.watched_at !== null`
- Формат `watched_at`: локализованная дата "DD.MM.YYYY"

## BR-09: Навигация назад
- Кнопка "Назад" на странице `/media/[id]` использует `router.back()`
- Если нет истории (прямой переход по URL) — `router.back()` ведёт на страницу 404 или не работает
  → дополнительная ссылка "Библиотека" как fallback (статичный href="/library")
