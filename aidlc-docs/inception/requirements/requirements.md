# Requirements Document

## Intent Analysis

- **User Request**: Создать личный трекер просмотра медиаконтента (фильмы/сериалы/аниме) без плеера — добавление тайтлов через TMDB API и отслеживание прогресса по сезонам и эпизодам
- **Request Type**: New Project (Greenfield)
- **Scope Estimate**: System-wide (полное веб-приложение)
- **Complexity Estimate**: Moderate — несколько взаимосвязанных сущностей, внешнее API (TMDB), аутентификация, деплой

---

## Technology Stack (зафиксированный)

| Слой | Технология |
|---|---|
| Frontend / Framework | Next.js 14+ (App Router) + TypeScript |
| Стилизация | Tailwind CSS + shadcn/ui |
| Backend / DB | Supabase (PostgreSQL + Auth + Storage) |
| Внешнее API | TMDB API (поиск и метаданные) |
| UI-тема | tweakcn (кастомизация shadcn/ui токенов) |
| UI-анимации | Aceternity UI (точечно: главная, карточки) |
| Деплой | Vercel (frontend) + Supabase Cloud (backend) |

## Functional Requirements

### FR-01: Аутентификация
- Пользователь входит в систему через email + пароль (Supabase Auth)
- Доступ ко всем страницам только после авторизации
- Поддержка выхода из аккаунта
- Сессии управляются через Supabase (JWT-токены)

### FR-02: Поиск и добавление контента через TMDB
- Пользователь может искать фильмы, сериалы и аниме по названию через TMDB API
- Результаты поиска отображаются в виде карточек с постером, названием и годом
- Пользователь выбирает тайтл из результатов — метаданные подтягиваются автоматически:
  - Название (оригинальное + локализованное)
  - Описание
  - Постер (URL из TMDB)
  - Год выпуска
  - Жанры
  - Тип: movie / tv / anime (anime — отдельный тип, определяется при добавлении)
  - Для сериалов/аниме: список сезонов и количество эпизодов в каждом (из TMDB)
- Добавленный тайтл сохраняется в коллекцию пользователя

### FR-03: Управление коллекцией
- Пользователь видит все добавленные тайтлы в виде сетки карточек с постерами
- Каждая карточка показывает: постер, название, тип, статус, прогресс (для сериалов)
- Пользователь может удалить тайтл из коллекции

### FR-04: Статусы просмотра
Каждый тайтл имеет один из статусов:
- `watching` — Смотрю
- `completed` — Просмотрено
- `planned` — Хочу посмотреть
- `dropped` — Брошено
- `on_hold` — Отложено

Пользователь может менять статус в любой момент.

### FR-05: Отслеживание прогресса (сериалы и аниме)
- Структура: сезоны + эпизоды (данные из TMDB)
- Для каждого тайтла отображается список сезонов с эпизодами
- Пользователь отмечает отдельные эпизоды как просмотренные (чекбокс)
- Поддержка "отметить весь сезон как просмотренный"
- Прогресс отображается: "S2E05 — 5/13 эпизодов" или "Сезон 2 из 3"
- Для фильмов прогресс не применяется — только статус
- При добавлении тайтла из TMDB сохраняется `runtime` (минуты):
  - Фильм: поле `runtime` из TMDB movie detail
  - Эпизод сериала: поле `runtime` из TMDB episode detail (точное время каждого эпизода)
  - Если TMDB не вернул `runtime` для эпизода — используется `episode_run_time[0]` сериала как fallback

### FR-10: Статистика просмотра
- Отдельная страница `/stats` с общей статистикой по коллекции пользователя
- **Общее затраченное время**: сумма `runtime` всех отмеченных как просмотренные эпизодов + фильмов со статусом `completed`
  - Отображается в формате: "X дней Y часов Z минут"
- **Разбивка по типу**: отдельно для фильмов / сериалов / аниме
- **Разбивка по статусу**: сколько тайтлов в каждом статусе (watching / completed / planned / dropped / on_hold)
- **Топ жанры**: топ-5 жанров по количеству тайтлов
- Статистика считается на клиенте из данных Supabase (без отдельного endpoint)

### FR-06: Оценка и заметки
- Пользователь может поставить оценку тайтлу: 1–10 (целое число или полузвёзды)
- Пользователь может добавить текстовые заметки (свободный текст)
- Оценка и заметки отображаются на странице тайтла
- Оба поля необязательны

### FR-07: Жанры
- Жанры подтягиваются из TMDB при добавлении тайтла
- Отображаются как теги на карточке и странице тайтла
- Используются для фильтрации

### FR-08: Поиск и фильтрация коллекции
- Поиск по названию внутри коллекции (мгновенный, без запросов к TMDB)
- Фильтры:
  - По статусу (watching / completed / planned / dropped / on_hold)
  - По типу (movie / tv / anime)
  - По жанру
  - По оценке (от X до Y)
- Сортировка: по дате добавления, по названию, по оценке
- Фильтры комбинируются (AND-логика)

### FR-09: Страница тайтла
- Детальная страница для каждого тайтла:
  - Постер, название, описание, жанры, год
  - Статус (выпадающий список для изменения)
  - Оценка и заметки (редактируемые inline)
  - Для сериалов/аниме: список сезонов с чекбоксами эпизодов

## Non-Functional Requirements

### NFR-01: Производительность
- Страница коллекции загружается < 2 сек при < 500 тайтлов
- Поиск по TMDB реагирует < 1 сек (debounce на 300 мс)
- Supabase запросы оптимизированы индексами на `user_id`, `status`, `type`

### NFR-02: UX / Отзывчивость
- Адаптивная верстка: работает на десктопе и мобильных
- Тёмная тема (тематика "кинотеатр" предполагает dark UI)
- Оптимистичные обновления для чекбоксов эпизодов (без задержки UI)

### NFR-03: Надёжность
- Все данные хранятся в Supabase (PostgreSQL) — персистентность гарантирована
- Row Level Security (RLS) в Supabase — пользователь видит только свои данные
- Обработка ошибок TMDB API (rate limits, недоступность)

### NFR-04: Деплой
- Vercel для Next.js (автодеплой из main-ветки)
- Supabase Cloud для БД и Auth
- Environment variables: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `TMDB_API_KEY`

### NFR-05: Дизайн-система — "Naruto grown-up"

**Концепция**: dark cinematic seinen — ночная элегантность, сдержанная восточная эстетика, тёплый amber-акцент на тёмном фоне. Вдохновение: зрелый Наруто, Jujutsu Kaisen, Blue Period.

**Цветовая палитра (tweakcn токены)**:
- Background: `#0D1117` (deep navy-black)
- Surface / Card: `#1E2A3A` (тёмный индиго)
- Border: `#2D3F55` (приглушённый стальной)
- Primary accent: `#E67E22` (янтарно-оранжевый)
- Primary hover: `#F39C12` (золотой)
- Text primary: `#F0F4F8`
- Text muted: `#8899AA`
- Destructive/dropped: `#C0392B`
- Success/completed: `#27AE60`

**Типографика**:
- Заголовки: Geist (или Inter с weight 700+)
- Body: Inter / Geist Mono для кодо-подобных элементов (номера эпизодов)

**Aceternity UI — точечное применение**:
- Главная страница: Spotlight или Beam effect на hero-секции
- Карточки тайтлов: CardSpotlight hover-эффект
- Страница тайтла: мягкое свечение (Glow) вокруг постера
- НЕ применять на: формах, таблицах, навигации — только декоративные акценты

**Общие UI-принципы**:
- Тёмная тема по умолчанию, light-тема вне scope MVP
- Постеры — ключевой визуальный элемент, UI не конкурирует с ними
- Скруглённые карточки (rounded-xl), плавные тени, без жёстких границ
- Статусы отображаются цветными badge-метками

### NFR-06: Масштабируемость фундамента
- Архитектура должна легко расширяться: добавление новых типов контента, импорт/экспорт, статистика
- Типы TypeScript для всех сущностей (shared types)
- Supabase schema с миграциями (не ручные изменения через UI)

## Data Model (высокий уровень)

```
users            (Supabase Auth — встроенная таблица)

media_items      — основная таблица тайтлов пользователя
  id, user_id, tmdb_id, type (movie/tv/anime)
  title, original_title, overview
  poster_url, release_year
  genres (jsonb array)
  status (watching/completed/planned/dropped/on_hold)
  rating (1-10, nullable)
  notes (text, nullable)
  created_at, updated_at

seasons          — сезоны сериала
  id, media_item_id, season_number, name, episode_count
  tmdb_season_id

episodes         — эпизоды
  id, season_id, episode_number, name
  runtime_minutes (integer, nullable)  -- из TMDB episode.runtime
  is_watched (boolean), watched_at (nullable)
  tmdb_episode_id

media_items      — добавить поле:
  runtime_minutes (integer, nullable)  -- для фильмов: TMDB movie.runtime
```

## Scope Boundaries (MVP)

### Включено в MVP
- Аутентификация (email/password)
- TMDB-поиск и добавление тайтлов
- Управление коллекцией (просмотр, удаление)
- Статусы просмотра
- Прогресс по сезонам/эпизодам
- Оценка и заметки
- Жанры из TMDB
- Поиск и фильтрация коллекции
- Статистика просмотра (общее время, разбивка по типу/статусу, топ жанры)
- Деплой на Vercel + Supabase Cloud

### Вне MVP (будущие итерации)
- Импорт/экспорт данных (CSV, JSON)
- Расширенная статистика (графики по времени, активность по дням)
- Списки/плейлисты
- Пользовательские теги (помимо жанров)
- Социальные функции
- Push-уведомления о новых сезонах
- Мобильное приложение

## Security Note

Security Extension отключён пользователем (Q10:B). Тем не менее, базовые меры применяются по умолчанию:
- Supabase RLS обеспечивает изоляцию данных на уровне БД
- Все API-ключи хранятся в переменных окружения (не в коде)
- Next.js middleware защищает роуты от неавторизованного доступа
